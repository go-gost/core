# The core library

`github.com/go-gost/core` defines all interfaces and shared types for the GOST tunneling toolkit. It has **no implementations** and **zero third-party dependencies** вҖ” just Go's standard library. All concrete types live in the [`x/`](https://github.com/go-gost/x) module.

## Interfaces

The module defines every component of the GOST proxy pipeline:

- **Service** вҖ” Wraps a listener + handler into a runnable server
- **Listener** вҖ” Accepts inbound connections (TCP, TLS, HTTP/2/3, QUIC, etc.)
- **Handler** вҖ” Processes inbound connections: authenticates, routes, forwards
- **Dialer** вҖ” Dials from one node to the next proxy hop
- **Connector** вҖ” Connects from the final node to the destination
- **Chain / Route / Transporter / Node / Hop** вҖ” Proxy chain routing and node selection
- **Router** вҖ” Top-level router combining chain, resolver, host mapping, retries, and timeouts
- **Registry[T]** вҖ” Generic typed registry for component registration
- **Selector / Strategy / Filter** вҖ” Generic load balancing and node selection

Supporting types: `Authenticator`, `Admission`, `Bypass`, `Resolver`, `HostMapper`, `Recorder`, `Observer`, `Ingress`, `SD` (service discovery), `Logger`, `Metrics`, `Limiter` (conn/rate/traffic), `Stats`, `Matcher`, and `Router` (OS-level).
